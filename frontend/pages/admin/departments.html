Department Management API
The Department CRUD API (GET/POST/PUT/DELETE /api/v1/departments/) is planned for a future module. The departments table is already present in the database and linked to complaints and users.
Department Data Model
ColumnTypeDescription
idInteger (PK)Auto-increment primary key
nameString(100)Unique department name
descriptionTextOptional description
created_atDateTime (UTC)Creation timestamp
Department Relationships
  • Each User can belong to a department via department_id
  • Each Complaint can be linked to a department via department_id
  • Deleting a department sets department_id = NULL on linked records (SET NULL)
